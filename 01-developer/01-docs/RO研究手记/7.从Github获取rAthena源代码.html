从Github获取rAthena源代码

2014年12月29日2015年3月15日 Sola丶小克 未分类

本文主要介绍如何从rAthena的代码仓库中下载最新的rAthena代码，为之后的编译和研究做准备。rAthena的代码寄存在Github上，项目地址为：https://github.com/rathena/rathena 请打开此链接。

打开项目地址后，点击右侧“HTTPS clone URL”如下图所示的的复制按钮：

这样你就会复制到一个类似：https://github.com/rathena/rathena.git 的地址，然后请在D盘或者其他你想要存放代码的地方，空白处单击鼠标右键，选择“Git 克隆”（如果没有此选项，请参照《在Windows系统中安装Git客户端》进行操作），会看到如下界面：

当你确认好保存目录后，就可以点击“确定”，开始获取代码。获取过程中根据你的网速不同，需要的时间也不同，请耐心等…

当你看到如下界面时，表示已经获取完毕，可以关闭了：

当克隆完毕后，大家可以在刚刚选定的磁盘下看到一个rathena文件夹，比如我的是在C:\rathena，并且这个文件夹的图标会有一个绿色的对勾：

这个绿色的对勾，表示这个目录里面的内容是否被你更改过，绿色表示从你克隆下来到现在，此文件是没被你修改过的。如果是红色感叹号，则表示这个目录里面有文件（如果是文件带红色感叹号那么表示这个文件本身）被你修改过。

当你修改过里面的内容的话，红色感叹号会出来看着还是挺烦人的，为了解决这个问题，请下载我提供的一个批处理文件：http://pan.baidu.com/s/1nthyXex 解压后将里面的“移除Git版本控制.bat”放到你克隆出来的目录下（我这里是C:\rathena）：

然后运行这个bat文件，根据提示操作，最终你会在你克隆出来的目录的同级目录下，看到一个名为“rathena_yyyymmdd”的目录（如果碰到360安全卫士提示是否放过的话，请勾选“不再提示”，并选择允许），如下图所示：

刷新几次后可以看到，这个目录是没有绿色对勾，也没有红色感叹号的，如果有绿色对勾和红色感叹号的话，那么请删掉它，重新执行bat文件。

建议大家将带绿色对勾的rathena目录，重命名为rathena_github，将被bat文件生成出来的rathena_yyyymmdd目录，重命名为rathena。以后我们的修改主要就在这个rathena目录上进行。
